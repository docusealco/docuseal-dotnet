# docuseal-dotnet

.NET SDK for the DocuSeal API. The entire SDK is generated by Fern
(fern-csharp-sdk): `required` record properties enforce required fields at
compile time, System.Text.Json serialization, multi-targeting
(net462/netstandard2.0/net8/net9), retries and timeouts built into
ClientOptions. Nothing is written by hand - this replaced an NSwag-generated
client with a hand partial.

## Architecture

- `src/Docuseal/` - generated, never edit by hand. Regenerate with
  `./generate-types.rb [spec-path-or-url]` (Node.js, Docker and ruby
  required; the Fern generator runs locally in Docker, no account). Building
  all target frameworks needs the .NET 9 SDK.
- `fern/` - Fern workspace config: generator version pin,
  `namespace: Docuseal`, `client-class-name: DocusealClient`.
- `Directory.Build.props` - NuGet package metadata (PackageId, Version,
  license); kept outside src/ so regeneration does not touch it.
- `generate-types.rb` fetches the spec in SDK mode (`?sdk=true`; the
  webhook drop, `/submissions/init` swap, tag strip and
  `<OperationId>Params` naming happen on the Rails side) and appends
  `PermanentlyDeleteTemplateAsync`/`PermanentlyDeleteSubmissionAsync` (same
  DELETE endpoints with `?permanently=true`; not expressible in OpenAPI
  next to the archive operations) into `DocusealClient.cs`/
  `IDocusealClient.cs` plus their Params records under `Requests/`.

## Source of truth

The OpenAPI spec is generated by `lib/api_spec/` in the docuseal-website
repository and served at https://console.docuseal.com/openapi.yml. Do not
fix type issues here - fix them in the spec and regenerate.

## Conventions

- Client surface is flat: `client.GetTemplateAsync(id)`,
  `client.CreateSubmissionAsync(params)`; method names from operationIds.
- Build: `dotnet build src/Docuseal` (local SDKs live in `~/.dotnet`).
- The Fern output also produces a `Docuseal.Test` project; the generate
  script copies only `src/Docuseal`. No tests yet.
